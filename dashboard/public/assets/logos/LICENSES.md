## Replacement contract

When an official mark is added:

1. Replace the SVG file in place. Keep the filename, viewBox `0 0 64 64`,
   and an `aria-label` matching the id.
2. Flip the row above from `placeholder` to the matching license string
   (`CC-BY-4.0`, `CC-BY-NC-4.0`, `MIT`, `Brand-Permission-Email`, etc).
3. Add the URL of the canonical asset and the date of retrieval.
4. If the brand requires attribution, append a `Required Attribution` row
   to the table and surface that string somewhere visible in the dashboard
   (typically the about / footer panel).

## Why placeholders, not bundled brand artwork

Shipping a third party's trademarked logo on a public dashboard without a
recorded grant is a copyright/trademark exposure. Brand-color letter marks
preserve enough visual identity for the mobile mindmap to be readable at a
glance while keeping the dashboard inside fair-use safe ground until each
brand mark is replaced under its declared license.

## Generator

Run `node dashboard/public/assets/logos/generate.mjs` from repo root to
re-emit every placeholder file from `CHAIN_MARKS` / `PROTOCOL_MARKS` and
refresh `manifest.json`. The generator is intentionally idempotent and
overwrite-safe; manual edits to a `placeholder`-status SVG will be lost on
the next run.
