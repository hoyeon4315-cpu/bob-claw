The blocker resolver is layered on top of `src/prelive/admission-remediation.mjs`: it keeps the existing whitelisted refresh parser and execution summary as the command-runner boundary, then adds stable blocker codes, recipe selection, retry/circuit state, and receipt-confirmed operational outcomes. Admission remediation remains the route/input refresh implementation; the resolver decides when a normalized blocker should reuse that path, queue capital-manager work, wait, or surface operator/code review without weakening policy, signer, cap, kill-switch, dev-lock, or payback authority.
